TFT_eSPI provides two complementary approaches to displaying text. The `print()` / `println()` stream interface inherited from Arduino's `Print` class advances a cursor automatically, while the explicit `drawString()`, `drawNumber()`, and `drawFloat()` family positions output using a configurable datum point. Font selection, colour, size, alignment, wrapping, and UTF-8 decoding are all controllable at runtime. Built-in fonts are numbered 1–8 (fonts 4, 6, 7, and 8 are Run Length Encoded), and GFX free fonts can be loaded with `setFreeFont()`.

## Method Summary

| Method | Signature | Returns | Description |
|---|---|---|---|
| `print()` | `print(val[, format])` | `void` | Stream text at cursor |
| `println()` | `println(val[, format])` | `void` | Stream text + newline |
| `drawString()` | `drawString(string, x, y[, font])` | `int16_t` | Draw string at position |
| `drawCentreString()` | `drawCentreString(string, x, y, font)` | `int16_t` | Centre-justified string |
| `drawRightString()` | `drawRightString(string, x, y, font)` | `int16_t` | Right-justified string |
| `drawChar()` | `drawChar(x, y, c, color, bg, size)` | `void` | Draw single char (GFX style) |
| `drawChar()` | `drawChar(uniCode, x, y[, font])` | `int16_t` | Draw Unicode char |
| `drawNumber()` | `drawNumber(intNumber, x, y[, font])` | `int16_t` | Draw integer |
| `drawFloat()` | `drawFloat(floatNumber, dp, x, y[, font])` | `int16_t` | Draw float value |
| `setTextColor()` | `setTextColor(color)` | `void` | Set foreground colour |
| `setTextColor()` | `setTextColor(fg, bg[, bgfill])` | `void` | Set fg + bg colours |
| `setTextSize()` | `setTextSize(size)` | `void` | Set size multiplier |
| `setTextFont()` | `setTextFont(font)` | `void` | Select built-in font |
| `setFreeFont()` | `setFreeFont(f)` | `void` | Load GFX free font |
| `setTextDatum()` | `setTextDatum(datum)` | `void` | Set alignment datum |
| `getTextDatum()` | `getTextDatum(void)` | `uint8_t` | Read alignment datum |
| `setTextPadding()` | `setTextPadding(x_width)` | `void` | Set overwrite padding |
| `getTextPadding()` | `getTextPadding(void)` | `uint16_t` | Read padding width |
| `setTextWrap()` | `setTextWrap(wrapX[, wrapY])` | `void` | Configure line wrapping |
| `textWidth()` | `textWidth(string[, font])` | `int16_t` | Measure string width |
| `fontHeight()` | `fontHeight([font])` | `int16_t` | Measure font height |

## print() / println() — Arduino Print Stream

```cpp
void print(val)
void print(val, format)
void println(val)
void println(val, format)
```

Inherited from the Arduino `Print` base class. Renders any printable value at the current cursor position and advances the cursor. `println()` appends a newline, moving the cursor to the left edge of the next line. This is the most convenient way to display dynamic data because the cursor advances automatically.

<ParamField path="val" type="any">
  The value to render. Accepts `String`, `char*`, `int`, `long`, `float`, `double`, etc.
</ParamField>
<ParamField path="format" type="int">
  Optional format specifier (e.g. `DEC`, `HEX`, `BIN`, or number of decimal places for floats).
</ParamField>

```cpp title="Streaming values to the display"
tft.setCursor(10, 20);
tft.setTextColor(TFT_WHITE, TFT_BLACK);
tft.setTextSize(2);
tft.println("Hello, World!");
tft.print("Value: ");
tft.println(42);
tft.print("Hex: 0x");
tft.println(255, HEX);
```

## drawString() — Draw String at Position

```cpp
int16_t drawString(const char *string, int32_t x, int32_t y, uint8_t font)
int16_t drawString(const char *string, int32_t x, int32_t y)
int16_t drawString(const String& string, int32_t x, int32_t y, uint8_t font)
int16_t drawString(const String& string, int32_t x, int32_t y)
```

Draws a string at `(x, y)` using the specified font or the current font. The position is interpreted relative to the current text datum set by `setTextDatum()`. Use this instead of `print()` when you need precise control over placement or alignment.

<ParamField path="string" type="const char* or String&">
  The text to render. Accepts both `char[]` and Arduino `String` types.
</ParamField>
<ParamField path="x" type="int32_t">X coordinate, interpreted via the current text datum.</ParamField>
<ParamField path="y" type="int32_t">Y coordinate, interpreted via the current text datum.</ParamField>
<ParamField path="font" type="uint8_t">Font number (1–8). If omitted the current font is used.</ParamField>

<ResponseField name="return" type="int16_t">
  Rendered string width in pixels.
</ResponseField>

```cpp title="Drawing a centred string"
tft.setTextDatum(MC_DATUM); // middle-centre alignment
tft.drawString("Hello!", tft.width() / 2, tft.height() / 2, 4);
```

## drawCentreString() — Centre-Justified String

```cpp
int16_t drawCentreString(const char *string, int32_t x, int32_t y, uint8_t font)
int16_t drawCentreString(const String& string, int32_t x, int32_t y, uint8_t font)
```

Draws a string horizontally centred at x-coordinate `x`.

<Warning>

</Warning>

<ParamField path="string" type="const char* or String&">Text to render.</ParamField>
<ParamField path="x" type="int32_t">X coordinate of the horizontal centre of the string.</ParamField>
<ParamField path="y" type="int32_t">Y coordinate of the top of the string.</ParamField>
<ParamField path="font" type="uint8_t">Font number (1–8).</ParamField>

<ResponseField name="return" type="int16_t">
  X coordinate at the end of the rendered text.
</ResponseField>

```cpp title="Centre-justified string (deprecated)"
tft.drawCentreString("Centred", 160, 10, 2);
```

## drawChar() — Draw Single Character

```cpp
// GFX / GLCD font overload
void drawChar(int32_t x, int32_t y, uint16_t c,
              uint32_t color, uint32_t bg, uint8_t size)

// Current font and colours
int16_t drawChar(uint16_t uniCode, int32_t x, int32_t y)

// Explicit font number
int16_t drawChar(uint16_t uniCode, int32_t x, int32_t y, uint8_t font)
```

Renders a single character. The first overload mirrors the Adafruit GFX interface and accepts explicit foreground/background colours and a pixel-doubling size. The second and third overloads accept a Unicode code point and use the current text colour settings.

<ParamField path="x" type="int32_t">X coordinate for the character top-left corner.</ParamField>
<ParamField path="y" type="int32_t">Y coordinate for the character top-left corner.</ParamField>
<ParamField path="c" type="uint16_t">Character code (ASCII or Unicode glyph).</ParamField>
<ParamField path="color" type="uint32_t">Foreground colour (overload 1 only).</ParamField>
<ParamField path="bg" type="uint32_t">Background colour (overload 1 only).</ParamField>
<ParamField path="size" type="uint8_t">Size multiplier (overload 1 only).</ParamField>
<ParamField path="uniCode" type="uint16_t">Unicode code point to render (overloads 2 & 3).</ParamField>
<ParamField path="font" type="uint8_t">Font number (overload 3 only).</ParamField>

<ResponseField name="return (overloads 2 & 3)" type="int16_t">
  Character width in pixels. Overload 1 returns `void`.
</ResponseField>

```cpp title="Drawing single characters"
tft.drawChar(10, 10, 'A', TFT_WHITE, TFT_BLACK, 2); // GFX style
tft.drawChar('H', 50, 10, 4);                        // font 4, Unicode point
```

## drawNumber() — Draw Integer

```cpp
int16_t drawNumber(long intNumber, int32_t x, int32_t y, uint8_t font)
int16_t drawNumber(long intNumber, int32_t x, int32_t y)
```

Draws a long integer at the specified position. The position is subject to the current text datum set by `setTextDatum()`. Compatible with `setTextPadding()` to erase previously displayed values when new values are shorter.

<ParamField path="intNumber" type="long">Integer value to display.</ParamField>
<ParamField path="x" type="int32_t">X coordinate.</ParamField>
<ParamField path="y" type="int32_t">Y coordinate.</ParamField>
<ParamField path="font" type="uint8_t">Font number. If omitted the current font is used.</ParamField>

<ResponseField name="return" type="int16_t">
  Rendered text width in pixels.
</ResponseField>

```cpp title="Drawing elapsed seconds right-aligned"
tft.setTextDatum(MR_DATUM);
tft.setTextPadding(60); // always clear 60px to erase old digits
tft.drawNumber(millis() / 1000, 200, 120, 4);
```

## drawFloat() — Draw Floating-Point Number

```cpp
int16_t drawFloat(float floatNumber, uint8_t decimal, int32_t x, int32_t y, uint8_t font)
int16_t drawFloat(float floatNumber, uint8_t decimal, int32_t x, int32_t y)
```

Draws a floating-point number with a specified number of decimal places. The position respects the current text datum.

<ParamField path="floatNumber" type="float">Float value to display.</ParamField>
<ParamField path="decimal" type="uint8_t">Number of decimal places to render (0–9).</ParamField>
<ParamField path="x" type="int32_t">X coordinate.</ParamField>
<ParamField path="y" type="int32_t">Y coordinate.</ParamField>
<ParamField path="font" type="uint8_t">Font number. If omitted the current font is used.</ParamField>

<ResponseField name="return" type="int16_t">
  Rendered text width in pixels.
</ResponseField>

```cpp title="Displaying a sensor reading"
float temperature = 23.75f;
tft.setTextDatum(ML_DATUM);
tft.drawFloat(temperature, 2, 10, 80, 4); // renders "23.75"
```

## setTextColor() — Set Text Colour

```cpp
// Transparent background (foreground only)
void setTextColor(uint16_t color)

// Opaque background
void setTextColor(uint16_t fgcolor, uint16_t bgcolor, bool bgfill = false)
```

Sets the text foreground colour. The second overload also sets a background colour painted behind each character glyph. When `bgfill` is `true` the background is extended across the full padding width set by `setTextPadding()`.

<ParamField path="color" type="uint16_t">Text foreground colour (transparent background overload).</ParamField>
<ParamField path="fgcolor" type="uint16_t">Foreground (text) colour.</ParamField>
<ParamField path="bgcolor" type="uint16_t">Background colour painted behind each character.</ParamField>
<ParamField path="bgfill" type="bool">If `true`, fill the full padding width with `bgcolor`. Default `false`.</ParamField>

```cpp title="Setting text colours"
tft.setTextColor(TFT_WHITE);              // white text, transparent background
tft.setTextColor(TFT_YELLOW, TFT_BLACK);  // yellow on solid black
```

## setTextSize() — Set Font Size Multiplier

```cpp
void setTextSize(uint8_t size)
```

Sets an integer pixel-doubling multiplier applied to all subsequent text rendering. Size 1 is native resolution; size 2 doubles both width and height, and so on.

<ParamField path="size" type="uint8_t">
  Size multiplier (1–7). Each step adds 8 pixels to the base character height: 1 = 8 px, 2 = 16 px, 3 = 24 px, etc.
</ParamField>

```cpp title="Using size multipliers"
tft.setTextSize(1); // native 8px height
tft.setTextSize(3); // 24px height (3× enlarged)
```

## setTextFont() — Select Built-In Font

```cpp
void setTextFont(uint8_t font)
```

Selects the active built-in font by number for all subsequent `print()` and `drawString()` calls. Fonts 1–3 are proportional GLCD bitmaps; fonts 4, 6, 7, and 8 are Run Length Encoded large fonts. Font 0 selects the GLCD default.

<ParamField path="font" type="uint8_t">Font number (0–8). Values outside the range of compiled-in fonts are ignored.</ParamField>

```cpp title="Selecting a built-in font"
tft.setTextFont(4); // large RLE font
tft.drawString("Big Text", 10, 10);

tft.setTextFont(2); // smaller proportional font
tft.drawString("Smaller", 10, 50);
```

## setFreeFont() — Select GFX Free Font

```cpp
void setFreeFont(const GFXfont *f = NULL)
```

Loads an Adafruit GFX-compatible proportional font. Pass `NULL` or call with no argument to revert to the built-in font. Requires `LOAD_GFXFF` to be set in `User_Setup.h`.

<ParamField path="f" type="const GFXfont *">
  Pointer to a `GFXfont` structure (typically a `PROGMEM` constant in a header from the `Fonts/` directory). `NULL` reverts to the built-in GLCD font.
</ParamField>

```cpp title="Using a GFX free font"
#include <Fonts/FreeSansBold12pt7b.h>

tft.setFreeFont(&FreeSansBold12pt7b);
tft.drawString("Free Font!", 10, 30);
tft.setFreeFont(NULL); // revert to built-in
```

## setTextDatum() — Set Text Alignment Datum

```cpp
void setTextDatum(uint8_t datum)
```

Sets the reference point used to position text. The `(x, y)` coordinate passed to draw functions is mapped to the selected datum point on the text bounding box. See the [Overview](/api/tft-espi/overview) page for the full list of datum constants.

<ParamField path="datum" type="uint8_t">
  A datum constant. Key values: `TL_DATUM` (0, default), `TC_DATUM` (1), `TR_DATUM` (2), `ML_DATUM` (3), `MC_DATUM` (4), `MR_DATUM` (5), `BL_DATUM` (6), `BC_DATUM` (7), `BR_DATUM` (8), `L_BASELINE` (9), `C_BASELINE` (10), `R_BASELINE` (11).
</ParamField>

```cpp title="Centring text on screen"
tft.setTextDatum(MC_DATUM);
tft.drawString("Centred", tft.width() / 2, tft.height() / 2, 4);
```

## setTextPadding() — Set Overwrite Padding Width

```cpp
void setTextPadding(uint16_t x_width)
```

Defines a pixel width that is always cleared to the background colour when drawing text. This prevents stale characters from remaining visible when a shorter string overwrites a longer one — a common issue with live sensor readouts.

<ParamField path="x_width" type="uint16_t">
  Padding width in pixels. The background fills from the end of the rendered string up to this total width. Set to 0 to disable.
</ParamField>

```cpp title="Using padding to erase old values"
tft.setTextColor(TFT_WHITE, TFT_BLACK);
tft.setTextDatum(ML_DATUM);
tft.setTextPadding(120); // always clear 120px of background
tft.drawString(String(sensorValue), 10, 50, 4);
```

## setTextWrap() — Configure Text Wrapping

```cpp
void setTextWrap(bool wrapX, bool wrapY = false)
```

Controls whether `print()` / `println()` wraps text at the right edge and optionally at the bottom edge of the display. Wrapping does not apply to the explicit `drawString()` family.

<ParamField path="wrapX" type="bool">If `true`, text wraps to the next line when reaching the right edge of the display.</ParamField>
<ParamField path="wrapY" type="bool">If `true`, text wraps back to the top when reaching the bottom edge. Default `false`.</ParamField>

```cpp title="Enabling horizontal wrapping"
tft.setTextWrap(true, false);
tft.setCursor(0, 0);
tft.print("This long string will wrap at the display edge automatically.");
```

## textWidth() — Measure String Width

```cpp
int16_t textWidth(const char *string, uint8_t font)
int16_t textWidth(const char *string)
int16_t textWidth(const String& string, uint8_t font)
int16_t textWidth(const String& string)
```

Returns the pixel width of a string if rendered with the specified (or current) font without actually drawing anything. Use this for layout calculations such as manual centering or drawing a background box before text.

<ParamField path="string" type="const char* or String&">The string to measure.</ParamField>
<ParamField path="font" type="uint8_t">Font number. If omitted the current font is used.</ParamField>

<ResponseField name="return" type="int16_t">
  String width in pixels.
</ResponseField>

```cpp title="Manually centring text"
int16_t w = tft.textWidth("Hello", 4);
tft.drawString("Hello", (tft.width() - w) / 2, 10, 4);
```

## fontHeight() — Measure Font Height

```cpp
int16_t fontHeight(uint8_t font)
int16_t fontHeight(void)
```

Returns the pixel height of the specified font, or the current font when called without arguments. Useful for computing line spacing in multi-line layouts.

<ParamField path="font" type="uint8_t">Font number. If omitted the current font is used.</ParamField>

<ResponseField name="return" type="int16_t">
  Font height in pixels.
</ResponseField>

```cpp title="Computing multi-line layout"
uint8_t font = 4;
int16_t lineH = tft.fontHeight(font);

tft.drawString("Line 1", 10, 10, font);
tft.drawString("Line 2", 10, 10 + lineH, font);
tft.drawString("Line 3", 10, 10 + lineH * 2, font);
```

## Complete Example

```cpp title="Text methods showcase"
#include <TFT_eSPI.h>
#include <Fonts/FreeSansBold12pt7b.h>

TFT_eSPI tft = TFT_eSPI();

void setup() {
  tft.init();
  tft.setRotation(1);
  tft.fillScreen(TFT_BLACK);

  // Stream-style printing
  tft.setTextColor(TFT_WHITE, TFT_BLACK);
  tft.setTextFont(2);
  tft.setCursor(5, 5);
  tft.print("Temp: ");
  tft.print(23.5f, 1);
  tft.println(" C");

  // Centred heading with RLE font
  tft.setTextDatum(TC_DATUM);
  tft.setTextColor(TFT_CYAN);
  tft.drawString("TFT_eSPI Demo", tft.width() / 2, 40, 4);

  // Right-aligned counter with padding
  tft.setTextDatum(MR_DATUM);
  tft.setTextColor(TFT_YELLOW, TFT_BLACK);
  tft.setTextPadding(80);
  tft.drawNumber(millis() / 1000, tft.width() - 5, 100, 6);

  // GFX free font
  tft.setTextDatum(TL_DATUM);
  tft.setTextColor(TFT_ORANGE);
  tft.setFreeFont(&FreeSansBold12pt7b);
  tft.drawString("Free Font", 5, 130);
  tft.setFreeFont(NULL);
}

void loop() {}
```
